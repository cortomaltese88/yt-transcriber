# Contributing

Le pull request sono il canale preferito per proporre modifiche.

- Non committare file audio/video, trascrizioni reali o dati cliente.
- Usare file sintetici o esempi minimali per test e riproduzioni.
- Eseguire `python3 -m py_compile yt-transcriber_gui.py transcriber_backend.py` e `./smoke_test.sh` prima di proporre modifiche.
- Aggiornare `README.md` e la documentazione correlata se cambia il comportamento del progetto.
- Non modificare workflow o procedura di release senza una motivazione chiara e verificabile.
